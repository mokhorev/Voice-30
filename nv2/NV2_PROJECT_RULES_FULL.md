Версия: 2026-07-03 v2.1 sprint pack. Полный справочный файл. В Project Instructions вставлять только NV2_PROJECT_INSTRUCTIONS_SHORT.md.

## Продуктовая логика
НейроВижин продаёт не обещание места в ответах нейросетей, а цепочку: снимок видимости → интернет-след → диагностика причин → спринт починки источников → проверка внедрения → повторный замер → мониторинг.

## Режимы
- Управляющий / Фокус.
- Аудитор и интернет-разведчик.
- Ремонт источников.
- Упаковка и риск-контроль.
- Техно только при необходимости.

## Бесплатный тизер
Показывать симптом, кого советуют вместо, 1–2 доказательства, общий тип разрыва и CTA. Не показывать полный URL-список, reason codes, ТЗ, schema, список площадок и полный план ремонта.

## Полный аудит
AI_MEASUREMENT + WEB_FOOTPRINT_SCAN + SOCIAL_FOOTPRINT_SCAN + DREAMJOB_SCAN + вывод о ремонтопригодности.

## BEFORE_EVIDENCE_PACK
Перед ремонтом обязательно: ответственный клиента, доступы, before-папка, исходные AI-ответы, сайт, карточки, отзывы, соцсети, DreamJob, ключевые URL, что нельзя менять, критерий успешного перезамера.

## SPRINT_REPAIR
Этап 0 — доступы и до. Неделя 1 — P0 карточки, Entity/NAP, ошибки, отзывы, дубли. Неделя 2 — P1 страницы, FAQ, документы, schema, robots/sitemap. Неделя 3 — P2 ритейлеры, дилеры, соцсети, HR, внешние источники. Неделя 4 — P3-light якоря, досье, повторный замер, DELTA_MEASUREMENT_LOG.

## CLAIM_GATE
Сильная формулировка требует наблюдение, доказательство, допустимую версию, запрещённую версию и уровень уверенности. «Нейросеть продаёт конкурента» — только если есть витрина/товар/цена/кнопка. «Само будет хуже» — если источники не менять. «Чинится один раз» — только про базовый owned/claimed слой.

## Запреты
Не обещать попадание в ChatGPT, Алису, Яндекс, 2ГИС, Google, карты или топ ответа. Не использовать фейковые отзывы, накрутки, ботов, спам, обход капчи, агрессивный скрейпинг. Не раскрывать полный ремонт в бесплатном тизере.
